📚 GitHubアップロード完全ガイド

初心者からプロまで対応！3つの方法を詳しく解説

🌐 方法1：ブラウザで直接アップロード（最も簡単）

✅ メリット

  • ソフトインストール不要
  • 操作が視覚的で分かりやすい
  • 初心者におすすめ

❌ デメリット

  • 大量ファイルの処理が面倒
  • 複雑な操作には不向き
1 GitHubアカウント作成・ログイン
  • https://github.com にアクセス
  • 「Sign up」でアカウント作成（無料）
  • メール認証を完了
ここにGitHubトップページのスクリーンショット
2 新しいリポジトリを作成
  • 右上の「+」アイコンをクリック
  • 「New repository」を選択
  • リポジトリ名を入力（例：ai-contest-guide）
  • 「Public」を選択（他の人に見せたい場合）
  • 「Add a README file」にチェック
  • 「Create repository」をクリック
リポジトリ作成画面のスクリーンショット
3 ファイルをアップロード
  • 「Add file」→「Upload files」をクリック
  • ファイルをドラッグ&ドロップ、または「choose your files」でファイル選択
  • HTMLファイルの場合、「index.html」という名前にすると便利
  • 下部の「Commit changes」にコメントを入力
  • 「Commit changes」ボタンをクリック

💡 ファイル名のコツ

index.html - メインページとして自動的に表示される
README.md - プロジェクトの説明文（マークダウン形式）

4 GitHub Pagesで公開（オプション）
  • リポジトリの「Settings」タブをクリック
  • 左メニューから「Pages」を選択
  • 「Source」で「Deploy from a branch」を選択
  • 「Branch」で「main」を選択
  • 「Save」をクリック
  • 数分後、「https://ユーザー名.github.io/リポジトリ名」でアクセス可能

🎉 GitHub Pages の魅力

あなたのHTMLファイルが無料でWebサイトとして公開されます！
URLを共有するだけで、誰でもあなたの作品を見ることができます。

💻 方法2：GitHub Desktop（中級者向け）

✅ メリット

  • 直感的なGUI操作
  • 複数ファイルの一括管理
  • 変更履歴が分かりやすい

❌ デメリット

  • ソフトのインストールが必要
  • 初期設定がやや複雑
1 GitHub Desktop をダウンロード・インストール
  • https://desktop.github.com/ からダウンロード
  • インストール後、GitHubアカウントでログイン
  • 名前とメールアドレスを設定
2 リポジトリを作成またはクローン
  • 新規作成の場合：「Create a New Repository on your hard drive」
  • 既存をクローンの場合：「Clone a Repository from the Internet」
  • ローカルパス（保存先フォルダ）を指定
  • 「Create repository」または「Clone」をクリック
3 ファイルを追加・編集
  • 指定したローカルフォルダにファイルをコピー
  • GitHub Desktop で変更が自動検出される
  • 変更内容を確認
📁 my-project/
├── 📄 index.html
├── 📄 style.css
├── 📄 script.js
└── 📄 README.md
4 コミット・プッシュ
  • 左下の「Summary」欄にコミットメッセージを入力
  • 「Commit to main」をクリック
  • 「Push origin」でGitHubにアップロード

🎊 完了！

ブラウザでGitHubリポジトリを確認すると、ファイルがアップロードされています。

⌨️ 方法3：コマンドライン（上級者向け）

✅ メリット

  • 高速で効率的
  • 自動化しやすい
  • プロ仕様の操作

❌ デメリット

  • コマンド覚える必要がある
  • 初心者には難しい
  • 間違いが起きやすい
1 Git をインストール
  • Windows: https://git-scm.com/ からダウンロード
  • Mac: Homebrewで brew install git
  • Linux: sudo apt-get install git
# インストール確認
git --version
2 Git の初期設定
# ユーザー名とメールアドレスを設定
git config --global user.name "あなたの名前"
git config --global user.email "your-email@example.com"

# 設定確認
git config --list
3 リポジトリを初期化・クローン

🆕 新規プロジェクトの場合：

# プロジェクトフォルダを作成
mkdir my-project
cd my-project

# Gitリポジトリとして初期化
git init

# GitHubの空リポジトリと接続
git remote add origin https://github.com/ユーザー名/リポジトリ名.git

📥 既存リポジトリの場合：

# GitHubからクローン
git clone https://github.com/ユーザー名/リポジトリ名.git
cd リポジトリ名
4 ファイルを追加・コミット・プッシュ
# ファイルをステージングエリアに追加
git add .
# または特定ファイルのみ
git add index.html

# コミット（変更を記録）
git commit -m "初回コミット: HTMLファイルを追加"

# GitHubにプッシュ（アップロード）
git push -u origin main
5 日常的な更新作業
# 変更状況を確認
git status

# 変更内容を確認
git diff

# 更新作業の定型パターン
git add .
git commit -m "ファイルを更新"
git push

⚠️ よくあるエラーと対処法

認証エラー： Personal Access Token を使用
permission denied： SSH鍵の設定が必要
merge conflict： ファイルの競合を手動で解決
